Computational Thinking 计算思维

计算机的基本结构首先是需要了解的，在当下的时代我们有许多种类的计算机，比如说：台式电脑（Desktop）、笔记本（Laptop）、平板电脑（Tablet）等等，他们的大小和基本功能不太一样，但他们的基本架构是差不多相同的。

计算机结构

他们都有CPU（Central Processing Unit）、RAM（Random Access Memory）和存储，即Storage（包括硬盘和内存卡），而这些就被称为计算机结构。

那么这个结构到底是怎么样的呢？就不得不提到计算机结构模型，插一句，现代计算机其实又被称为冯·诺依曼结构计算机，是的，当代计算机就是以冯·诺依曼结构为基础设计的， 回归原题，计算机结构实际上遍被称为冯·诺依曼结构。

冯·诺依曼结构（The Von Neumann Model）

  • 中间那个巨大的长方形是计算机本身：
    • 每个计算机都有一个CPU，其中包含又一个控制单元（Control Unit）和一个算数逻辑单元（ALU，即，Arithmetic Logic Unit)，而最下面有一个内存单元（Memory Unit），它是用来存储数据和指令的。
      • 控制单元（Control Unit）：管理程序和应用如何运行的，它分配系统资源，并在系统组件之间进行协调。
      • 算术逻辑单元（ALU）：一个处理数据的“大型计算器”。
        形象一点，如果将CPU比作一个大家族，那么控制单元就是一个这个家族的大管家，算术逻辑单元勉强可以算作这个家族的幕僚，是诸葛亮型人物。
      • 内存单元（Memory Unit） 会与CPU进行数据和指令的交换，也就是他们之间的箭头。通常而言，内存单元指的是RAM（随机存取存储器），暂时存储一些临时程序和数据。
  • 我们通过输入设备（Input）将数据和指令输入计算机，比如摄像机、扫描仪、键盘、写字板等。
  • 在处理数据时，计算机可以在输出设备（Output）上显示结果或执行一些其他任务，而输出设备是用来接收计算机数据的输出的，比如显示器、打印机、绘图仪、扬声器等等。

What & Why Java

Java是什么

Java是一门面向对象的编程语言，开发者为James Gosling（詹姆斯·高斯林）。

  • Java特点：面向对象编程（OOP）：Object-Oriented Programming

Java作为一门超受欢迎的编程语言，本身是由Sun微系统公司研发的，刚开始Java还不叫Java叫做Oak，只不过后来Oracle收购了Sun。目前Java由Oracle对Java语言进行更新维护。 Java之所以受欢迎，有一点非常吸引人：

  • Java特点：平台独立性（Platform Independence）
    • 这意味着，Java语言可以在许多地方，从电脑端到移动端皆可运行，而不需要像其他编程语言一样还需要对代码进行修改。
      Java生态：火星车（Mars rover）、手机（Cell phones）、蓝光光盘（blu-ray disc）、网站服务端（Web server）、超级计算机（Supercomputing）
    • 这也就产生了一个说法，写就写一次，我到处运行（Write Once, Run Everywhere）。

Java的优势所在（Advantages）

  1. Higher cross- functionality and portability （更高的跨功能和可移植性）
    写就写一次，我到处运行。
  2. OOP, distributed, supports multithreading, offers multimedia and network support （面向对象的、分布式的、支持多线程的、支持多媒体和网络的）
  3. mature, therefore more stable and predictable（一门成熟的语言，因此更加稳定和可预测）
  4. large active user community and support available（拥有大量活跃的用户社区和可用的支持）
  5. Java programs are compiled independent of platform in bytecode language（Java程序是用字节码语言独立于平台进行编译的-平台独立性）
    这种语言允许相同的程序在任何安装了JVM的机器上运行(run on any machine that has a JVM installed)
  6. tons of frameworks（java有大量框架）

OOP Concepts Java中的OOP概念

  • 首先要明确的是OOP指的是Object-Oriented Programming（面对对象编程），上述这张图便是OOP的概念体系图，那么这张图上面单词都是是什么意思呢？
    • object 对象
    • class 类
    • inheritance 继承
    • polymorphism 多态性
    • abstraction 抽象
    • encapsulation 封装 面向对象编程（OOP）的主要目的是实现真实世界的东西，例如对象、类、抽象、继承、多态等。 OOP是一种范式或者说一套方法论，使用类和对象来设计程序，他提供了以上6个概念以简化程序开发和维护的过程。 接下来针对这六个概念简单解释：

Object 对象

  • 对象这个概念是从现实世界抽象过来的，凡是有状态（state）和行为（behavior）的东西都被称为对象（object）。
    比如一只修猫咪就是一个对象，它有名字、颜色等状态，还有吃东西、犯傻等行为。

OK然后我们回到编程中来，对象（object）可以定义为一例类（Class）。 既然对象要被使用，那么它就需要被定位，因此对象包含一个地址来明确它的位置，并在内存中占用一些空间。 对象可以在不知道彼此数据或代码细节的情况下进行通信，可以接收信息和返回响应，就像是两个人可以发微信也可以在QQ上匿名聊天。

Class 类

  • 对象（Object）的集合被称为类（Class），比如一群狗，比如有高个的在狗叫的狗、有肥肥的躺在地上的狗，他们被集合在一起，就被称为一类狗。
    类在内存中不占据空间。

Inheritance 继承

我们什么时候会提到继承呢？Yes，继承家产，我们继承家产的时候是从父辈身上继承的。

  • 同理我们在编程中也有父对象、子对象的概念，当一个对象（这个时候它属于一个子对象 Child Object）获得了父对象（Parent Object）的所有家产，也就是父对象的所有属性（state）和行为（behavior）时，这种状况便被称为继承（inheritance）。
    实际上继承时指的是一个对象基于另一个对象构建，被继承的类被称为超类（superclass），继承的类被称为子类（subclass）。

继承可以减少一些重复的代码，让代码变得具有可复用性（resuability），它被用来实现运行时多态性（polymorphism）。

Polymorphism 多态性

这个词听起来非常高大上，但解释起来也很简单。

  • 如果一个任务以不同的方式执行，就称之为多态性（polymorphism）。例如，用不同的方式说服客户；画一些东西，例如形状，三角形，矩形等。
    在Java中，我们通过方法重载（method overloading）和方法覆盖（method overriding）来实现多态性。 补充一下，方法（method）指的是你对对象（object）做的操作。

另一个例子是说话，人说人话，猫说猫话，狗说狗话等等，这些也是多态。

Abstraction 抽象

我们每天都会打电话，但我们不知道内部是如何处理通话/通信的，因为电话很抽象，电话内部细节和它的功能如何运作都被隐藏起来了，我们只需要会使用电话就行，这可比了解它的原理简单多了。

  • 由此，我们可以知道，隐藏内部细节和显示功能称为抽象（abstraction），抽象意味着用简单的东西来表示复杂，在Java中，抽象是指用对象（object）、类（class）和变量（variable）等简单的东西表示更复杂的底层代码和数据。
    在Java里，我们使用抽象类（abstract class）和接口（interface）来实现抽象。

Encapsulation 封装

吃药的时候，我们经常会吃一种胶囊，胶囊里面包裹着不同的药物，这就是封装。

  • 相似地，我们将代码和数据包装在一起，成为一个单元（或者说放一个胶囊里），这个行为就是封装（encapsulation）。

想想之上我们是不是说过，对象（object）的集合是类（class），一个java类就是封装的一个很好的例子。

封装是用于在面向对象编程中实现抽象的技术。封装用于对类成员和方法的访问限制。
  • 封装可以视作一种保护手段，就像是胶囊要把药物包裹起来不让药物倾泄和接触外界一样，封装将类中的数据私有化，保证类本身数据和代码的安全，然后我们只能通过公共方法（public methods）来访问他们。

补充的一些概念

除了以上六个概念之外，还有一些是会经常在面向对象设计中使用到的：

  • Coupling 耦合
  • Cohesion 内聚
  • Association 关联
  • Aggregation 聚合
  • Composition 组合

JDK/JVM/JRE

  • JDK（Java Development Kit Java开发包）
    • Java开发包是Java环境的核心组件，提供编译、调试和执行Java程序所需的所有工具、可执行文件和二进制文件。
  • JVM（Java Virtual Machine Java虚拟机）
    • Java虚拟机是Java编程语言的核心。运行Java程序时，JVM负责将字节码转换为特定于机器的代码。JVM依赖于平台，提供了内存管理、垃圾回收、安全等java核心功能。
  • JRE（Java Runtime Environment Java运行环境）
    • Java运行环境是JVM的实现。它提供了一个执行java程序的平台。JRE由JVM、java二进制文件和其他类组成，可以成功地执行任何java程序。
